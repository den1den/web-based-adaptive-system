Define the Event Condition Access Rules or the Adaptation Rules

To generate rules that define the adaptation behavior associated with the application just save the application, using button .
The Graph Author Tool will automatically generate two files named "example.gaf" and "example.aha". You can see the files in the "Author Files Tab" in AMT.

The ".gaf" extension is what is used by the Graph Author Tool to recreate the graph(ical) presentation. To open an already saved aplication double click on the ".gaf" file.

Commit the saved application to the running AHA! system by using . Only when edits are "committed" do they affect end-users that log in to the application. You can safely edit an application and save results a number of times. You only "commit" the application when you are confident that the structure is right and that end-users should start using it.
