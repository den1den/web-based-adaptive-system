The AHA! Configuration File

Not everything that is configurable in AHA! can be controlled through the configuration menu for the server manager. In the directory "WEB-INF" (a standard name for the Tomcat server) there are two configuration files: web.xml and ahaconfig.xml. The web.xml file is created automatically through the initial configuration process and should never be changed. It contains a pointer to the ahaconfig.xml file which has all AHA!-specific parameters. Although it is probably best to leave all parameters as they are, there may be reasons to change the following parameters:

  • "mysqlhost": This is set to "localhost" by default. It can be changed to separate the machine running the mySQL database from the machine running Tomcat and the AHA! servlets. Spreading the load may improve performance. (We have not tried this, as our server (aha.win.tue.nl) with dual Athlon MP2000+ is still fast enough for our applications, including the on-line AHA! tutorial.)
  • "mysqldatabase": This is the name of the mySQL database, optionally created during the initial AHA! configuration.
  • "mysqlpassword": This is the password for the mySQL database, optionally created during the initial AHA! configuration. Please note that this password is stored as cleartext and therefore the WEB-INF directory or ahaconfig.xml file should be kept secure!
  • "XMLROOT": This is the directory used to store the XML files when no mySQL database is used. As this directory includes a part for the user model files it may grow quite large. Therefore it may become necessary to move this directory to a different disk. (This should also help performance.)
