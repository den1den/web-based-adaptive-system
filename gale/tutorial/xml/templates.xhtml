Graph Author Templates

In the graph author tool you can create concepts and concept relationships of different types. The list of available types is based on templates. All files for authoring tools can be found in "/author/authorfiles" (set by the "graphauthorfiles" parameter in the AHA! configuration file). The file "templatelist.txt" lists the concept templates that exist in the "templates" directory and the file "crtlist.txt" lists the concept relationship templates that exist in the "crt" directory. The graph author checks these lists and downloads the templates. A concept template defines a structure of attributes, a concept type and unary concept relationships. A template consists of two parts (files):

  • The file with suffix ".author" contains definitions used by the graph author to determine how to display the concept relationships of the given type. The graph author only needs the information from this file (and the structure created by the author) to produce its graph output file.
  • The file with suffix ".aha" contains definitions for the translation of the concept relationships of the given type to AHA! event-condition-action rules. The graph author only needs the information from this file (and the structure created by the author) to produce the file with concepts and rules that you can also create using the concept editor.

Because of the strict separation of concerns in these two files, and the output the graph author creates from them, the ".author" files can also be used without the ".aha" files to create concept relationship graphs without translation to AHA!, and the ".aha" files can be used by different authoring tools to translate concept relationship graphs to AHA! rules without the graph author.

When creating new concepts in the Concept Editor a "defaultConcept.xml" file is used to create an initial set of attributes and rules. The attributes and rules can of course be changed in the concept editor.
