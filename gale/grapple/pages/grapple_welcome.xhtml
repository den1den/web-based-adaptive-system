GRAPPLE Tutorial

The GRAPPLE project aims at delivering to learners a Technology-Enhanced Learning (TEL) environment that guides them through a life-long learning experience, automatically adapting to personal preferences, prior knowledge, skills and competences, learning goals and the personal or social context in which the learning takes place. The same TEL environment can be used/accessed at home, school, work or on the move (using mobile/handheld devices). GRAPPLE includes authoring tools that enable educators to provide adaptive learning material to the learners. Authoring includes creating the conceptual structure of a subject domain, linking the concepts to resources (learning material), defining pedagogical rules to guide learning processes, and creating the pedagogical structure of a course. To ensure the wide adoption of adaptation in TEL GRAPPLE works with Open Source and commercial Learning Management Systems (LMSs). A distributed User Modeling Service architecture (GUMF) helps end-users to stay in control of their user profile while at the same time allowing them to use the profile to get personalized access to learning applications offered through different LMSs by different organizations.

The main target group of this tutorial are authors of adaptive courses, working in an academic or corporate setting, where learning is supported or facilitated through an LMS. In this tutorial we describe the authoring of adaptive courses, and focus on five parts of the GRAPPLE environment that are used for the authoring and delivery process:

  • The LMS needs to be set up in such a way that courses are linked to their adaptive learning material, and that results of some of the tests and assignments are communicated to GUMF.
  • The GRAPPLE Authoring Tool (GAT) is used by teachers to create the conceptual and educational structure of courses and to link concepts to resources.
  • The GRAPPLE Adaptive Learning Environment (GALE) delivers adaptive learning material to the learner. It is a very generic and extensible adaptation engine that can also be used stand-alone (without other GRAPPLE components).
  • GRAPPLE User Model Framework (GUMF) is a common (distributed) storage space for user-related information. It can store information that is used by just one component but it is mostly used as a common store that enables an LMS to exchange User Model (UM) information with GALE.
  • The GRAPPLE Event Bus (GEB) makes all GRAPPLE components communicate with each other. GEB needs to be installed and configured and all components need to be told (configured) where to find GEB.

We advise you to first study the overall architecture/infrastructure of GRAPPLE before studying the role of the individual components.

Note: Throughout this tutorial we use a large number of acronyms (such as LMS, GAT, GUMF, GALE, GEB...). If you are unsure about their meaning you can mouse over the first occurrence of the term on a page and the full name will appear as a tooltip. We do assume that you are familiar with some Web Technology terms such as HTML, HTTP, XML, XHTML, CSS, DOM, SOAP, URL, URI, SPARQL, SeRQL, etc. and do not offer explanations or tooltips for these terms.
