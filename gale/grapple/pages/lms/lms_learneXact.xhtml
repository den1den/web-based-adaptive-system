Using Learn eXact in GRAPPLE

GRAPPLE (and specifically the GRAPPLE Conversion Component module) can be installed only on eLex v. 2009 1 3617 150 (and possibly later versions to come). GRAPPLE Deliverable 5.2b contains detailed installation instructions that go beyond the scope of this tutorial.

The process of creating a course and (multiple-choice) tests in Learn eXact is not influenced by the use of GRAPPLE. However, the way a course is linked to the adaptive learning material and the communication of test results to GUMF both are GRAPPLE-specific. To use GRAPPLE you need to link Learn eXact to . A file, called "Web.config" contains the address of the event bus, like: 

<configuration>
   <appSettings>
      <add key="gebListenerService" value="http://www.grapple-project.org/GrappleEventBus/gebListenerService?wsdl"/>
      <add key="eventGEBListenerService" value="http://www.grapple-project.org/GrappleEventBus/eventGEBListenerService?wsdl"/>
   </appSettings>
</configuration>
Note that the URL given here is just an example and needs to correspond to where GEB really is.
Note also that the GRAPPLE software repository does not contain the actual Learn eXact software extensions as they are proprietary.

Now we consider the creation of a course and a multiple-choice test. In Learn eXact the author of a course can be different from the teacher. The author can create the course in the and then publish it and hand it over to a teacher. We now consider the teacher Paul who wishes to create a "Solar System" course which uses a "Sun" quiz. To this end the course is marked as a "structured" course to which content can be added using the content editor:



  • The course in Learn eXact is linked to adaptive course material by first searching for existing adaptive courses (served through GALE) and then selecting the appropriate one (in this case "Milkyway").



    The adaptive course material is defined as a "URL", so learners get access to it by simply clicking on a link. The adaptive learning material (served by GALE) is presented in a separate browser window. The actual URL is part of the response to the getCourses request.
  • A multiple-choice test is created using the authoring tool "Exact Packager" and saved as a SCORM package. It is then incorporated into Learn eXact's ditigal repository by "publishing" it. By making the test "public" it can be used by others. Paul adds the "Sun" quiz to the learning material (as shown in the figure of the content editor). When learners take the test the results are stored in GUMF and can be used by GALE for adaptation of the learning material.
