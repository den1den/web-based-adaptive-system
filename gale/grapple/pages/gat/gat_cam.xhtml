Creating a (Pedagogical) Course Model

The Course Adaptation Tool us used to define the pedagogical structure of a course. (The result is sometimes referred to as the Conceptual Adaptation Model or CAM and sometimes also as the Pedagogical Course Model.) Whereas the represents the conceptual structure of the topic domain of a course, the CAM connects concepts or sets of concepts with each other based on pedagogical or presentation goals. In the DM a relationship connects two concepts (like A is "parent" of B). In the CAM we can create relationships between an arbitrary number of sets of concepts. The placeholders for the related (sets of) concepts are called "sockets". Throughout the description of the Course Adaptation Tool we will use some pedagogical relationship types (or PRTs), but we have a separate description of the standard pedagogical relationship types that are part of the GRAPPLE distribution. As an example when some set A of concepts should be studied before you can understand the concepts of set B we create a prerequisite relationship between set A and set B. The pedagogical relations (may) have adaptive behavior associated with them. For instance, with the prerequisite between A and B links to concepts of set B may be hidden until all the concepts of A have been studied.

In addition to the previous method, pedagogical relations can also be copied and pasted. Copying a relation is done by right-clicking on a relation node and selecting “Copy Relation”. Then, pasting is done by right-clicking in empty space on the canvas and choosing “Paste Relation”. This creates a new instance of the same PRT, including possibly modified colurs and shapes, but with empty sockets. In the current Course Editor all PRT instances are separate. When A is a prerequisite of B and B is a prerequisite of C the graphical display will show the two sockets for "B" separately. However, the tool does show in a visual way that sockets of different relationships have concepts in common:

A relationship can also be edited by selecting "Edit" from the context menu:



You can change the color and shape of the presentation of a relationship. However the most important functionality is that you can change parameter values. A prerequisite may require "80%" knowledge of the first concept(s), but this number is actually a parameter that can be changed, for instance to reduce the required level of knowledge.

Other editing functions are fairly straightforward: deleting a relationship, deleting the concepts in a socket (currently you can only delete all concepts at once), saving the course model to the database or exporting it to a file.

When a course (pedagogical structure) is ready you can select "Deploy" from the "File" menu. This sends the course (CAM) to GALE to enable learners to start studying the course.
