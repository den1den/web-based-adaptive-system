The PluginModule (<plugin> tag)

The <test> is a special-purpose tag used to include a link to a special function in a page. The header of each page of the tutorial contains: 

  <gale:plugin>
    <gale:name>logout</gale:name>
    <gale:linkdescription>Logout
    </gale:linkdescription>
  </gale:plugin>
This generates a link that looks like: logout Logout and that logs you out when you click on the link. (We do not suggest that you try this now.)

When a plugin needs additional parameters they can be specified using the "param" subelement. An example: 

<gale:plugin>
  <gale:name>export</gale:name>
  <gale:linkdescription>Export concept (hierarchy)
  </gale:linkdescription>
  <gale:param name="root"
    value="gale://gale.tue.nl/cam/GRAPPLE"/>
</gale:plugin>
This results in the following link: export Export concept (hierarchy) . When you click on this link GALE produces a textual representation of the GALE for the named concept and all other concepts starting with this name. In practice this would export the DM for this entire GRAPPLE tutorial. The format is named "GDOM" for GALE Domain Model.

In the advanced GALE topics we explain how to create new plugins. Here we describe the standard plugins that are included in the GRAPPLE (GALE) distribution:

LogoutPlugin (logout)
This plugin logs you out. It generates an HTML page that displays a message indicating that you have been logged out. Note that this only works in stand-alone mode. If you are logged in using Shibboleth (the IdPLoginManager) the logout plugin will log you out of GALE but you remain logged in on Shibboleth because Shibboleth has no logout mechanism. (You are logged out of Shibboleth when you exit the browser.)
FormPlugin (form)
This plugin allows for GALE code execution based on information entered in an HTML form. The plugin expects the code that should be executed in an HTTP parameter. The code can use the form parameters in the regular way (i.e. by calling gale.req().getParameter(...)).
ExportPlugin (export)
As shown above, this exports part of the DM as a GDOM file that is sent as content to the browser. This plugin uses an additional parameter called 'root' to select part of the domain model. All concepts that start with the value of this parameter are included.
PasswordPlugin (password)
This plugin displays a password change form that allows the user to change his/her password. When using Shibboleth this plugin has no effect.
UpdatePlugin (update)
Allows the direct import of a .cam file. The parameter 'cam' is used as a URL to locate the .cam file. A .cam file is a saved using the "export" functionality of .
MCPlugin
This plugin is used by multiple choice tests to produce an evaluation report (score). There is no name to call this plugin directly from within a page using the <plugin> tag.
ExecPlugin (exec)
This plugin is used by GALE adaptive links to execute code upon following a link. There is no name to call this plugin directly from within a page using the <plugin> tag.
AjaxPlugin (ajax)
This plugin is used by the ajax processor to indicate subsequent document retrieval. It returns a new document only if something has changed.
