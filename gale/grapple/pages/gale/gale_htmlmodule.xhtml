The HTMLModule (and <html> tag)

GALE is mostly used to perform adaptation to HTML or XHTML files. These files all start with an <html> tag. This tag is followed by the <head> and <body> parts of the document. These two parts are optional. GALE needs to add to elements to the <head> element:

  • <link rel="stylesheet" type="text/css" href="${home}/gale.css" />
    This ensures that the standard GALE CSS file is used, which defines the colors to be used in the link adaptation.
  • <base href="http://host:port/gale/concept/gale://gale.tue.nl/cam/GRAPPLE/gale_adaptingresources/${home}/grapple/pages/gale/gale_adaptingresources.xhtml" />
    This (with the real name and number instead of the "host" and "port" parts) ensures that relative links from with a page result in the correct reference.

It may seem strange that the manipulation to the <head> element is done by a module that handles the <html> tag and not the <head> tag. However, because the <html> tag is mandatory and the <head> tag is optional a module that manipulates the <head> tag would not be triggered if the <head> tag were absent from a page.
