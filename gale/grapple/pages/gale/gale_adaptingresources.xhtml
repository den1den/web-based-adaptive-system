Adapting Content and Links in

The XMLProcessor in GALE traverses the DOM tree of the resource (page) to be presented. Each time it encounters an element (tag) for which a "Module" has been defined it lets that Module perform adaptation to the element. Typically the Module replaces the element with some different content, which must be valid XML.

We describe each Module with associated tag separately. The Modules/tags are:

The IfModule (<if> tag)
Used for the conditional presentation of some (in-line) content.
The ObjectModule (<object> tag)
Used for the conditional inclusion of a separate resource (file).
The AdaptLinkModule (<a> tag)
Used for adaptive link annotation or hiding.
The VariableModule (<variable> tag)
Used for inserting the value of a attribute or the value of an expression over the UM.
The AttrVariableModule (<attr-variable> tag)
Like the VariableModule, but used to insert the value into an attribute of an XML tag.
The ForModule (<for> tag)
Used for iterating over a list of objects.
The CountModule (<count> tag)
Generates the length of a list of objects.
The MCModule (<test> tag)
Used for generating a multiple-choice test.
The PluginModule (<plugin> tag)
Used for generating a link to a plugin for special functions (like e.g. logging out).
The ViewModule (<view> tag)
Used for generating a presentation over the , for instance the accordion menu you see on the left in this tutorial.
The HTMLModule (<html> tag)
Used for replacing the top HTML tag by a structured layout.
