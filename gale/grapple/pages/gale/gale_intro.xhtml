Introduction to Adaptive Learning and

GALE is the successor of the Adaptive Hypermedia Architecture AHA! that was developed at the Eindhoven University of Technology. GALE takes objectives of AHA! to a higher level: it is designed to be as flexible and extensible as possible so as to give authors the freedom to choose between a large number of adaptive methods and techniques. Some basic characteristic of GALE are:

  • Courses consist of concepts and relationships, together forming a Domain Model (DM). This corresponds to the structure of a DM as produced using the DM editor of . If you wish a course to have an accordion menu "on the left" as in this tutorial you should use "parent" relationships between concepts to create a concept hierarchy.
  • For each concept GALE keeps track of the user's (learner's) interaction with that concept. This is done by keeping information in User Model (UM) attributes. Typical attributes would be knowledge (how much does the learner know about the concept), visited (how often did the learner access the concept) and suitability (does the learner have the required foreknowledge to understand the concept).
  • With each concept GALE associates behavior or adaptation rules. A rule may for instance say that when the concept is accessed the visited counter should be incremented, the knowledge value should be increased, the knowledge value of a parent concept may also be increased, the suitability of some other concepts may become "true", etc. In literature the term Adaptation Model (AM) is often used for this model of concepts with adaptation rules, but within GALE we consider this as part of the DM. (We do realize that this is a misnomer as a DM is typically a structure independent of the pedagogy of the course, whereas in GALE DM and adaptation are mixed. So we sometimes write DM/AM to indicate we need the adaptation that is part of the GALE DM.)
  • Writing individual adaptation rules mentioned above is too cumbersome for authors. The editor that is part of GAT lets you "draw" pedagogical relationships between concepts from which a compiler automatically generates the adaptation rules for GALE. The implementation of the rules uses template adaptation rules. The GRAPPLE distribution comes with a sufficiently rich set of predefined templates so you do not need to study the tool to create your own templates.
  • For each concept of DM you typically write one page of learning material. (You can write more pages and have GALE select one to display based on UM values.) XHTML is used for the pages, with some GALE-dependent extensions represented as "gale tags". The <gale:if> tag is used to conditionally include a bit of text, for instance a short additional explanation the learner may need. Frequently occurring pieces of information may be conditionally included using the <gale:object> tag so you need to write the text fragment only once.
  • Links in a GALE course always refer to concepts (never to pages directly). The <gale:a> tag is used to define an adaptive link. By default links are presented in one of three possible ways: "good" (blue) which means the link is recommended, "neutral" (purple) which means the link is recommended but the target was already visited, and "bad" (black) which means the link is not recommended (the target is not suitable). By not underlining bad links with black text they become effectively hidden within the text. In the menu on the left you see another way of adapting links: A green ball indicates the link is recommended, a white ball indicates a visited recommended link and a red ball indicates a non-recommended link.
  • Apart from "access" to concepts other forms of interaction can also be associated with adaptation. GALE can serve XHTML forms to let users set and change certain UM variables, and it can serve and evaluate multiple-choice tests.
  • In GRAPPLE it is important to give application/course developers maximum freedom in deciding not only how the adaptation works but also in the design of the look and feel of their application. Although (to keep authoring simple) GALE comes with predefined presentation and adaptation templates, it is also reasonably easy for authors to create their own look and feel, either by changing the configuration (for a static different look and feel) or through a "layout PRT" (for a possibly adaptively changing look and feel).
The remainder of the GALE part of this tutorial part is aimed at authors who wish to know how the adaptation behavior described above really works and at authors who wish to change one or more aspects of this behavior. If you don't like the tags use for adaptation or you want to add more tags with different behavior, you can. If you want to add a new type of assessment (in addition to the multiple-choice tests), you can. If you want to change the layout, you can. If you want to process other data types (besides XHTML or even XML) you can. Almost every aspect of GALE's behavior can be changed either through a configuration or through the addition of new code for which GALE already contains the necessary hooks.
