AHA! Supporting Software Environment

AHA! consists of Java Servlets. While it may be possible to make AHA! work with any environment that supports Java Servlets, we recommend the environment described here. We only describe the server-side requirements. There are no client-side requirements other than a fully functional operating system and WWW browser.

Operating System

In theory any operating system can be used if it supports a Java 2 environment. The AHA! team has (successfully) tested AHA! on different versions of Microsoft Windows (up to Windows XP) as well as on Linux (Redhat versions 7 and 8 and Fedora core 4, 5 and 6).

Java Environment

AHA! requires the use of Sun's Java 2 environment (J2SE). It works with the Java SDK versions 1.4 and newer. (Older versions do not work, and alternatives like Kaffe do not work either.) Successive releases of the Java SDK have different new features (new classes and/or methods), and deprecate and abandon previously existing features. This may cause problems with newer Java SDK versions. We will include support for newer Java SDK versions in newer AHA! versions. The Java SDK can be downloaded from http://java.sun.com/j2se/downloads.html. Note that you must install the SDK not just the JRE (which is the runtime environment only). Compiling AHA! is part of the installation procedure and thus requires the SDK.

Web Server

AHA! needs a Webserver that can call Java Servlets. While several such Webservers exist they are all configured in different ways. We recommend the use of the Tomcat webserver. Although officially obsolete using Sun's JWSDP still makes perfect sense because it offers an easy way to configure Tomcat. We have successfully tested AHA! with versions 1.5 and 2.0 of JWSDP. When installing JWSDP you need to download and install an adapted TomCat server separately. JWSDP can be downloaded from http://java.sun.com/webservices/downloads/webservicespack.html. The adapted TomCat server can be downloaded from http://java.sun.com/webservices/containers/tomcat_for_JWSDP_1_5.html. (AHA! itself does not use any JWSDP features. It runs on a "plain" Tomcat, but the version that comes with JWSDP makes the configuration easier.)

Database System

The use of a database with AHA! is optional. It does not seem to help the performance, but it does allow you to query the database with user profile information. Currently AHA! only works with the Open Source mySQL database system, that can be downloaded from www.mysql.org. AHA! will work without problems without installing mySQL (except that the options for using mySQL will not work). MySQL support works with Microsoft Windows as well as Linux. (We do not test it frequently though.) Note that in order for AHA! to work with mySQL you need to create the appropriate mySQL user, (which you can choose through the AHA! configuration tool but which is not created on the mySQL server through the AHA! configuration tool).
