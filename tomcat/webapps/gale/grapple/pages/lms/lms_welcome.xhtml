The role of an in GRAPPLE

A Learning Management System (LMS) is used (by an institute or company) to organize the learning process. Learners can enroll in courses (or be enrolled by management). The LMS and GRAPPLE are connected for two types of functionality:

  • The learner can directly access the adaptive learning material that is served through an instance of . Because both the LMS and GALE are connected to the same Shibboleth service the learner is already identified (and authorized) when clicking through from the LMS to GALE.
  • The learner can perform tasks and assignments of which the results are communicated to . The results can be transferred to GALE and influence the adaptation of the learning material.

The following user events have been identified. LMSs can send data (parameters) from these events to GUMF through . The data is mapped to the IMS LIP standard and encapsulated into Grapple Statements. For types of events listed below we link to a list of parameters an LMS can associate with the event. (More detailed information can be found in GRAPPLE Deliverable 5.2b).

  1. Access to a course: it identifies the moment when the user accesses a course
  2. Tests/quizzes: the user has completed a test
  3. Registration: the user logs on the first time to the LMS integrated with GRAPPLE
  4. Student enrollment: the student is enrolled in a new course
  5. User login: the user logs on to the LMS integrated with GRAPPLE
  6. Role change: the user has changed his role (e.g. from author to teacher or from teacher to learner)
  7. Learning activity change: a particular learning activity has been changed
  8. Learning activity addition: a particular learning activity has been added
  9. Learning activity removal: a particular learning activity has been removed
  10. Access to a learning activity: it identifies the moment when the user access a learning activity.

Connecting the LMS to GRAPPLE is slightly different for each LMS, but consists of similar steps. The GRAPPLE software repository contains "LMSconnectors" for each of the supported LMSs. Whether an LMS sends events (see above) to GEB/GUMF or not and which data about these events is communicated can be configured in the LMS, again in different ways depending on the LMS. We have an explanation for five different LMSs. You need to study only the connection and communication for the LMS you will be using. (Detailed technical information can be found in GRAPPLE Deliverable 5.2b and GRAPPLE Deliverable 7.5.) The supported LMSs are:

  • Moodle
  • Sakai
  • Claroline
  • Learn eXact
  • Clix
