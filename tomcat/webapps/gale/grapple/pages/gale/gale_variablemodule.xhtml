The VariableModule (<variable> tag)

Through the <gale:variable> tag you can include and in a page. The tag accepts two different parameters:

  • When the tag is used with the attribute name the parameter refers to UM information. For instance: 
        <gale:variable name="#visited" />
    prints the number of visits to the current concept. (Actually, it only counts visits that happened when the concept was suitable.) For the current concept this is . If you do "reload" you will see that the number gets incremented.
  • When the tag is used with the attribute expr the result of an expression in GALE code. For instance, the following code: 
        <gale:variable expr="${->(parent)?type}" />
    when included in this page, prints "" (without the quotes). So the "type" property of the parent of the current concept is "".
    The following code: 
    <gale:variable expr="gale.req().getRequestURL()" />
    results in: and illustrates the use of method calls in GALE code. It shows the URL that was used in the current request. It should match what the location field of the browser displays.
    The following code: 
    <gale:variable expr="${->(parent)[0]?event}"/>
    results in: , which is the adaptation code of the parent concept.
