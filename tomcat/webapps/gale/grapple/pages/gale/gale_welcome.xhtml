GALE: The GRAPPLE Adaptive Learning Environment

GALE is the Adaptive Learning Environment of GRAPPLE. It can be used stand-alone or in combination with other GRAPPLE components. This tutorial mainly describes the large number of features (and interfaces) used by authors and designers in order to configure the adaptation and presentation to their own desires.

We have a short gentle introduction to introduce the possibilities of an adaptive learning environment. If you are mainly wondering what an adaptive learning environment can do for you, and less interested in the technical details of how things really work, this may be all you need.

In order to understand GALE we have an architecture overview of GALE, showing the components and how they interact throughout the process of handling a request for a concept. We give a description of the main conceptual structures ns GALE, namely the domain model, the adaptation model and the .

The process of adaptation is described in detail. GALE supports content adaptation, link adaptation and presenation adaptation. These three adaptation methods are spread over three functional parts, one for locating resources (given a concept as reference), one for adapting the content of resources (including the links within that content), and adapting the presentation using layout and generated views on the link structure.

Most aspects of adaptation in GALE depend on the evaluation of expressions and the execution of statements over the GALE . We explain this GALE Code in detail, including how this code is used by expert authors to write code for s.

This tutorial has an advanced section describing the internal communication between GALE components, the communication with other GRAPPLE components and the many ways in which GALE can be extended with new functionality. Most readers can safely skip the advanced section.

All the information about GALE in this tutorial is taken from the GRAPPLE Deliverable 1.3c.
