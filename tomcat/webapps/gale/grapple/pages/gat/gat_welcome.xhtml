Authoring Courses for GRAPPLE

GRAPPLE is intended to support learning based on the concept of courses. This corresponds to the most common way in which education is organized in schools, universities and in company training. Creating (authoring) a course generally consists of three steps:

  • The topics (in GRAPPLE terminology these are concepts) of the course are determined. This may start with an unordered list, but is typically (later) organized into a hierarchy. The structure of the topic domain of the course is called the Domain Model (DM) and is defined using the Domain Model Tool.
  • A reading or study order is determined for the topics of the course. In a traditional course one single sequence of topics is chosen which is used during lectures and/or is the order in which the topics appear in a course text (and its table of contents). GRAPPLE aims at using (web-based) hypertext or hypermedia for a course text. A rich link structure connects many topics in a seemingly random way, based on semantic relationships between topics. However, not every possible sequence of following links to navigate through the course text is suitable for learning: some topics need to be studied before some other topics can be learned and understood. To offer guidance to the learner a structure of pedagogical relationships is created, typically consisting of so called prerequisites, and these are used to adapt the link structure on the fly. The Course Adaptation Tool is used to define the pedagogical structure of a course (which is sometimes called Conceptual Adaptation Model or CAM for historical reasons).
  • For each course topic (or concept) some learning material needs to be written. In GRAPPLE the most commonly used content type for learning material is XHTML, an XML-compliant version of HTML. (HTML can be used as well, and is internally converted to XHTML before being used further.) However, any XML-based format can be used with . GRAPPLE does not include an editor for the content. Any editor capable of creating XML documents can be used for this.
After logging in on the GRAPPLE Authoring Tool (GAT) the following welcome screen is presented:



Actions can be initiated through the menus, buttons in the toolbar or by clicking on any of the large colored sections of the welcome screen.

When creating courses you might occasionally find that you need a pedagogical relationship that is not provided by GRAPPLE. There is a Pedagogical Relationship Types Editor that lets you create new pedagogical relationship types or PRTs. Most authors/teachers will not need to use this tool and therefore it is hidden in the welcome screen and the menus. When you select Tools->Advanced Mode the welcome screen and menus change and you get access to the Pedagogical Relationship Types editor.
